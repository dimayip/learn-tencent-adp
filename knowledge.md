# 腾讯云智能体开发平台（Tencent Cloud ADP）知识库

> 本文档整理了腾讯云智能体开发平台（Tencent Cloud Agent Development Platform，简称 Tencent Cloud ADP）的核心知识点，按主题分类组织，面向云计算行业从业者，帮助快速掌握产品概念、能力、架构和实践。共 140 个知识点。

---

## 一、产品概述与定位

1. **产品定义**：腾讯云智能体开发平台（Tencent Cloud ADP）是面向企业的 AI 应用开发平台，基于 LLM 能力，帮助企业快速搭建智能体应用。
2. **产品前身**：ADP 由"大模型知识引擎（LKE）"于 2025 年 5 月 20 日全新升级而来。
3. **英文全称**：Tencent Cloud Agent Development Platform，缩写 ADP。
4. **目标用户**：既适合业务团队（产品/运营/客服）零代码搭建，也适合技术团队通过 API/SDK 深度集成。
5. **核心价值**：从配置、开发、调试、评测、发布到运营，提供一站式的智能体全生命周期管理。
6. **典型应用**：智能客服、企业知识助手、经营分析助手、营销文案生成、数据分析助手等。
7. **产品定位**：可控、可运营的生产级 AI 能力底座，支持从 PoC 试点到规模化落地。
8. **底层支撑**：依托腾讯混元大模型、腾讯优图实验室的 OCR 与文档解析技术。
9. **开放生态**：支持主流大模型与自建模型一键导入；全面兼容 MCP 协议。
10. **开源关联**：自研 Multi-Agent 判断与运行机制可参考开源项目 YouTu-Agent。

---

## 二、应用模式（四种核心模式）

11. **Claw 模式**：拥有独立工作空间，可自主写代码、调用 Skills/插件，像随时在线的"数字同事"，适合数据分析、内容生成。
12. **标准模式（Standard）**：用于搭建单个智能体应用，从简单问答起步，按需补充知识库和插件。
13. **Multi-Agent 模式**：适合"一个 Agent 无法完成、需团队协作"的复杂任务，由多个子 Agent 自动分工。
14. **单工作流模式（Workflow）**：把固化的业务流程拖拽编排成可复用流水线，适合工单、审批等。
15. **模式选择原则**：简单问答用标准；复杂决策用 Multi-Agent；标准流程用 Workflow；面向非技术用户的复杂任务用 Claw。
16. **Multi-Agent 协同方式**：支持自由转交（Handoff）、工作流编排、Plan-and-Execute 三种模式。
17. **应用包导入导出**：四种模式都支持以"应用包"形式导出/导入，便于跨账号迁移。
18. **应用模板**：平台预置教育、传媒、医疗、金融等多行业模板，开箱即用。

## 三、平台架构与组成

19. **平台分层**：交互层（对话/UI）、智能体层（Agent/Workflow/RAG）、能力层（模型/工具/知识库）、基础层（COS/计算/安全）。
20. **企业（Enterprise）**：一个腾讯云主账号下默认创建 1 个企业，作为虚拟组织的最高层级。
21. **工作空间（Workspace）**：企业内的资源隔离单元，用于团队/项目级的资源与权限隔离。
22. **应用（Application）**：在工作空间下创建的智能体实例，是用户使用的最终产品形态。
23. **知识库（Knowledge Base）**：存储企业私域文档、问答对、网页内容的集合，是 RAG 应用的底座。
24. **共享知识库**：可在多个应用间复用的公共知识库，避免重复构建。
25. **超级管理员**：主账号默认拥有最高权限，可管理企业、工作空间、用户和所有资源。
26. **工作空间角色**：包括空间创建员、空间管理员、应用开发者、知识管理员等多角色。
27. **权限解耦**：平台端权限分"功能权限"和"数据权限"，支持精细化授权。
28. **应用端权限**：发布出去的应用还可单独配置终端用户的访问与消息记录权限。

---

## 四、智能体核心组件

29. **Agent（智能体）**：具备目标理解、推理决策、工具调用、记忆管理能力的 LLM 驱动实体。
30. **Prompt（提示词）**：定义 Agent 角色、能力、风格、约束的指令；平台提供提示词模板。
31. **思考模型**：负责意图识别和路由决策，影响理解准确度。
32. **生成模型**：负责最终回答生成，影响语言质量与专业度。
33. **变量（Variable）**：在对话/工作流中传递和复用数据的容器，分系统变量和自定义变量。
34. **长期记忆**：让 Agent 跨会话记住用户偏好、历史信息，提升个性化体验。
35. **会话（Session）**：单次连续对话的上下文窗口，由 SessionId 标识。
36. **意图识别**：识别用户输入对应的业务意图，用于路由到不同处理分支。
37. **拒答问题**：可配置敏感、违规问题的统一拒答话术，保障内容安全。
38. **澄清询问**：当用户表述不清时，Agent 自动反问以获取必要信息。

---

## 五、模型广场（Model Marketplace）

39. **预置模型**：平台内置混元、优图精调、DeepSeek 等十余种模型，开箱即用。
40. **DeepSeek 满血版**：内置 DeepSeek-R1/V3 671B 全尺寸模型，支持深度思考。
41. **混元大模型**：腾讯自研通用大模型，支持中文场景的高质量对话与生成。
42. **优图精调模型（youtu-mrc）**：腾讯优图针对知识问答场景精调，命中知识时回答更精准。
43. **TI-ONE 自定义模型**：可一键同步腾讯云 TI-ONE 平台部署的自训练模型在线服务。
44. **第三方 OpenAI 兼容模型**：支持接入符合 OpenAI 协议的第三方/自建模型。
45. **Embedding 模型**：自研 2B 级向量模型，C-MTEB 基准测试达 SOTA 水平。
46. **Reranker 模型**：基于 LLM 的重排模型，支持 8K+ 长文本，提升检索精度。
47. **模型路由**：可针对思考、生成、Embedding 等不同节点配置不同模型，实现成本/效果平衡。
48. **模型计费**：不同模型按 Token 用量分别计费，可在用量看板查看明细。

## 六、知识库与 RAG

49. **知识库三大内容**：文档（Doc）、问答对（QA）、数据库表（结构化数据）。
50. **支持文档格式**：pdf、doc/docx、ppt/pptx、txt、markdown、html 等 10+ 种，单文档最大 200MB。
51. **网页爬取**：支持输入 URL，由平台自动爬取文本和图片构建知识库。
52. **文档解析**：基于优图 OCR，可识别图文混排、表格、公式，结构化抽取。
53. **智能结构化解析**：自动识别标题层级、段落、列表，输出结构化的知识片段。
54. **文档切分（Chunking）**：把长文档切成 chunk 以适配检索；可设置切分大小与重叠。
55. **解析切分干预**：自动解析效果不佳时，运营可手动调整切分边界。
56. **文档对比任务**：支持新旧版本文档对比，自动识别变更点，便于知识库更新。
57. **问答对（QA）**：直接以"问题-答案"形式录入，命中后高优先级返回，适合 FAQ 场景。
58. **从文档生成问答**：上传文档后，平台可用大模型自动抽取候选 QA。
59. **数据库知识源**：支持接入结构化表格，配合 Text-to-SQL 实现表格问答。
60. **知识检索（RAG）**：把用户问题转向量，检索知识库相似片段，注入 Prompt 上下文。
61. **多路召回**：同时使用向量检索和关键词检索，结果融合后送入 Reranker。
62. **GraphRAG（知识图谱 RAG）**：构建知识图谱，支持跨实体、跨段落的关系推理。
63. **Agentic RAG**：由 Agent 决策"何时检索、检索什么、是否补充检索"，多轮自适应。
64. **知识库 Schema**：为知识库定义字段元数据（来源、标签、时间），便于过滤检索。
65. **知识库标签**：给文档/QA 打标签，支持按标签划分适用范围、做权限隔离。
66. **不满意回复闭环**：用户点踩后，运营可查看上下文，迭代优化知识库。

## 七、工作流（Workflow）核心节点

67. **工作流定义**：由"开始 → 节点链 → 结束"组成的可视化业务流程编排。
68. **开始/结束节点**：分别定义全局输入与最终输出，是流程的入口与出口。
69. **回复节点**：向用户输出文本/卡片消息，可放在流程中间或末尾。
70. **大模型节点**：在流程中调用 LLM 完成生成、改写、总结等任务。
71. **大模型意图识别节点**：用 LLM 判断用户意图，选择不同分支。
72. **大模型知识问答节点**：在流程中执行 RAG，结合知识库给出答案。
73. **知识检索节点**：仅做检索不做生成，命中片段作为变量供后续节点使用。
74. **参数提取节点**：从用户对话中抽取关键参数（订单号、日期等）。
75. **条件判断节点**：根据变量值走不同分支，类似 if/else。
76. **循环节点 / 批处理节点**：对集合做循环或并行批处理，加速运行。
77. **代码节点**：执行 Python/JavaScript 自定义代码，处理数据转换、计算等。
78. **工具节点**：调用已配置的外部 API/插件。
79. **数据库节点**：直连业务数据库执行查询/写入。
80. **Agent 节点**：在工作流中嵌入子 Agent，把决策能力交给智能体。
81. **Widget 节点**：输出可视化卡片/表单，丰富前端展示。
82. **变量聚合/赋值/转换节点**：处理工作流中的变量编排与类型转换。
83. **单点调试**：大部分节点支持单点调试，加速开发迭代。
84. **工作流异步调用**：通过 API 异步触发工作流执行，长任务不阻塞调用方。

## 八、Multi-Agent 多智能体协作

85. **Multi-Agent 适用场景**：单一 Agent 能力不足、需分工协作的复杂任务。
86. **自由转交（Handoff）**：主 Agent 自主决定把对话交给哪个子 Agent。
87. **工作流编排协同**：用 Workflow 串联多个 Agent，按预定流程协同。
88. **Plan-and-Execute**：先由 Planner 拆解任务计划，再由 Executor 逐步执行。
89. **知识库检索 Agent**：专门负责 RAG 检索的子 Agent，被其他 Agent 按需调用。
90. **Agent 高级设置**：包括 Agent 描述、技能列表、最大迭代次数、超时设置等。

---

## 九、连接器、工具与 Skills

91. **连接器（Connector）**：对外部系统（API、数据库、SaaS）的统一抽象，是工具的载体。
92. **工具（Tool）**：可被 Agent/Workflow 调用的原子能力，比如"查天气""发邮件"。
93. **预置插件**：平台内置 150+ 高质量插件，覆盖搜索、图像、音视频、办公、数据等。
94. **自定义插件**：可通过 OpenAPI/Schema 把任意 HTTP API 接入平台作为工具。
95. **MCP 协议支持**：全面兼容 Model Context Protocol，可接入 MCP 生态工具。
96. **联网搜索插件**：内置联网搜索能力（如 youtu-mrc-standard），让模型获得实时信息。
97. **网页解析插件**：解析任意网页内容为结构化数据供 Agent 使用。
98. **Skills（技能）**：把一组 Prompt、工具、知识打包为可复用技能，类似"专家模板"。

---

## 十、Widget 可视化卡片

99. **Widget 定义**：富交互前端组件，让 Agent 输出超越纯文本，可点击、输入、可视化。
100. **Card 卡片**：结构化展示一组信息，常用于商品、订单、文章等。
101. **Form 表单**：收集用户结构化输入，配合 Input/Select/Checkbox 等组件。
102. **Chart 图表**：基于 ECharts 渲染折线、柱状、饼图等可视化。
103. **Markdown 组件**：渲染富文本与代码块。
104. **Widget Action**：定义按钮/卡片点击后触发的动作（发消息、跳转、调工具）。
105. **Widget 创建方式**：模板创建、代码创建（JSON）、自然语言生成、导入。
106. **ADP-Widget SDK**：前端 SDK，可在自有 H5/小程序里渲染 Widget 内容。

## 十一、应用发布与集成

107. **发布渠道概览**：支持微信公众号、微信客服、微信小程序、企业微信、企微机器人、钉钉机器人。
108. **企业微信应用发布**：把 Agent 嵌入企业微信侧边栏或自建应用。
109. **API/SDK 集成**：提供 RESTful API、HTTP SSE、WebSocket 三种对话接入方式。
110. **ADP-Chat-Client**：官方开源前端客户端项目，可作为快速集成范本。
111. **首字时间**：流式响应首字延迟 < 1 秒，保证良好用户体验。
112. **流式输出（SSE）**：边生成边返回，提升对话体感。
113. **WebSocket 双向通信**：支持事件推送、打字机效果、客户端控制中断。

## 十二、计费与套餐

114. **套餐订阅模式**：按月/按年订阅，包含 PU（Processing Unit）资源额度，超出按量计费。
115. **PU 资源**：平台统一计费单位，模型调用、检索、文档解析等都折算为 PU。
116. **后付费模式**：开通"后付费开关"后，用量超出额度按账单结算。
117. **免费体验额度**：首次开通赠送一定量联网搜索、网页解析、知识库字符额度。
118. **知识库字符包**：按字符数购买的知识库存储扩容包。
119. **共享并发包**：购买并发能力包，应对高峰期请求。
120. **扣费顺序**：体验资源包 → 付费资源包 → 后付费，逐级扣减。
121. **云部署计费**：选择独立部署（容器/轻量服务器）时按计算与存储资源计费。

---

## 十三、运营、监控与安全

122. **应用评测**：内置裁判模型、规则、代码三种评估方式，可批量对比 Prompt/模型版本。
123. **数据报表-资源看板**：实时查看 Token、PU、并发、存储等资源消耗。
124. **数据报表-业务看板**：查看会话量、活跃用户、热点问题、点赞点踩等业务指标。
125. **告警管理**：配置阈值告警（调用失败率、延迟），通过 Webhook 推送到企微/钉钉/飞书。
126. **操作日志**：记录企业内所有关键操作，可审计追溯。
127. **资源管控**：可在企业层为子账号/工作空间设置用量上限，防止超支。
128. **审批中心**：发布、删除等敏感操作可走审批流程，控制风险。
129. **内容安全审核**：内置接入腾讯云内容安全，对输入输出做合规审核。
130. **多租户隔离**：企业、工作空间数据强隔离，避免越权访问。
131. **回答可解释**：每条回答附带命中文档片段、相似度评分、检索路径，便于问题定位。

---

## 十四、API 与开发者

132. **API 协议**：基于腾讯云 API 3.0，使用 TC3-HMAC-SHA256 签名方法 v3。
133. **核心 API 模块**：空间、应用、Agent、会话、模型、知识库、发布等管理 API。
134. **对话接口**：分 HTTP SSE 和 WebSocket 两种，长会话推荐 WebSocket。
135. **临时密钥（STS）**：上传文件时通过临时密钥保障安全，避免长期密钥泄漏。
136. **应用 AppKey 与 SecretKey**：应用唯一标识与鉴权凭据。

---

## 十五、最佳实践与典型场景

137. **智能客服场景**：意图识别 + RAG 回答，支持转人工兜底。
138. **企业知识助手**：内部文档构建知识库，员工自助问答提升效率。
139. **经营分析助手**：连接数据库 + Text-to-SQL，让业务自助查数据。
140. **OpenClaw 金融助手**：基于 Claw 模式打造能查股票、写研报的金融 Agent。
